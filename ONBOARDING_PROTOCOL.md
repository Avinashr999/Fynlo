# Fynlo Onboarding Protocol

**Version:** 1.0
**Last updated:** 2026-05-25
**Audience:** Future contributors, AI agents picking up the project after a long gap, future-you returning after a break.

---

## 0. Why this document exists

If you walk away from Fynlo for 6 months and come back, the cost of remembering everything is high. If someone else (a collaborator, an AI agent in a fresh session) tries to contribute, the cost of explaining everything is high. This protocol exists so onboarding is reproducible.

---

## 1. First 30 minutes — orientation

Read these documents in order:

1. `README.md` — what is Fynlo, who's it for
2. `GOVERNANCE_README.md` — how the project is governed
3. `PROJECT_STATE_FOR_AI.md` §0 — required reading for AI agents
4. `PROJECT_STATE_FOR_AI.md` §1-§4 — architecture, conventions, dependencies
5. `PROJECT_STATE_FOR_AI.md` latest journal entries — what's happened recently

After this you should know:
- What Fynlo does
- What language / framework / DB it uses
- What's been built and what's planned
- Where the rules live

## 2. Next 60 minutes — environment setup

### 2.1 Tools

| Tool | Version | Purpose |
|---|---|---|
| Android Studio | Latest stable (Hedgehog or newer) | Primary IDE |
| JDK | 17 | Required by AGP 9.x |
| Git | Any recent | Version control |
| Android SDK | API 36+ | Build target |
| ADB | Bundled with SDK | Device testing |

### 2.2 Clone

```powershell
cd C:\Users\user\AndroidStudioProjects
git clone https://github.com/Avinashr999/Fynlo.git
cd Fynlo
```

### 2.3 Secrets

Files NOT in git that you need:

- `secrets/keystore.properties` — keystore credentials (get from `BACKUP_PROTOCOL.md` location)
- `secrets/fynlo-release.jks` — signing keystore (same)
- `local.properties` — auto-generated by Android Studio on first open; contains `sdk.dir` path
- `app/google-services.json` — IS in git (currently); should remain so or be regenerated from Firebase Console

### 2.4 Build

```powershell
./gradlew :app:assembleProdDebug
```

If this succeeds, environment is OK. If it fails, see §3 (troubleshooting).

### 2.5 Run tests

```powershell
./gradlew testProdDebugUnitTest
```

All tests should pass on master.

### 2.6 Run on device

Connect device via USB (OnePlus CPH2767 or any Android 8+ device), enable USB debugging, then:

```powershell
./gradlew :app:installProdDebug
```

App launches. Sign in with Google. You should see the Dashboard.

## 3. Common setup issues

### 3.1 KSP InvalidPathException

Symptom: `./gradlew :app:generateProdBaselineProfile` fails with `Illegal char <?>` exception.

This is a known KSP 2.x bug on Windows. Workaround: skip baseline profile generation when not needed. Use the existing baseline profile in `app/src/main/baseline-prof.txt` instead.

See `PROJECT_STATE_FOR_AI.md` §5.3 for details.

### 3.2 Gradle daemon timeouts

Symptom: builds hang, especially first build.

Solution:
```powershell
./gradlew --stop
./gradlew clean :app:assembleProdDebug --no-daemon
```

### 3.3 Hilt / KSP errors

If `:app:kspProdDebugKotlin` fails:
1. Check KSP and Kotlin versions match (per `libs.versions.toml`)
2. Run `./gradlew clean` and rebuild
3. Invalidate Android Studio caches (File → Invalidate Caches → Just Restart)

### 3.4 Firebase / google-services.json missing

If `google-services.json` is missing:
1. Firebase Console → Project Settings → General → Your apps → Android app
2. Download `google-services.json`
3. Place in `app/` directory

## 5. Making your first change

Follow the standard workflow:

### 5.1 Pick a task

From `UX_AUDIT_2026-05-25.md` — find a P0 or P1 cluster that's open. Avoid P3 until P0 ship-blockers are done.

### 5.2 Branch

```powershell
git checkout master
git pull
git checkout -b fix/c01-recalculate-preserves-paid
```

Branch naming: `{type}/{cluster}-{short-description}`. Types: `fix`, `feat`, `chore`, `docs`, `test`.

### 5.3 Write test first (for data-integrity work)

Per `AI_AGENT_PROTOCOL.md` §6.

### 5.4 Make change

Edit code. Run tests. Verify on device.

### 5.5 Run lint

```powershell
./gradlew detekt
./gradlew :app:lintProdRelease
```

Fix any new violations.

### 5.6 Commit

```powershell
git add -A
git commit -m "fix(c01): preserve paid field through recalculate

Closes audit points #67, #68, #69.
Adds FynloRecalculateTest to prevent regression."
```

### 5.7 Push + PR

```powershell
git push -u origin fix/c01-recalculate-preserves-paid
```

Open PR on GitHub. The PR template will load — fill every section.

CI runs (per `.github/workflows/checks.yml`). All checks must pass.

### 5.8 Merge

After review and CI green, merge. Update `UX_AUDIT_2026-05-25.md` to mark cluster as Done. Update `PROJECT_STATE_FOR_AI.md` journal.

## 6. AI agents specifically

If you're an AI agent (Claude, Cursor, Copilot, etc.) being onboarded to Fynlo:

1. Read `AI_AGENT_PROTOCOL.md` IN FULL before touching anything
2. Produce the session-start acknowledgement block (§1.1 of that protocol)
3. Don't propose code without identifying which cluster it addresses
4. Follow the output format in §7 of that protocol
5. If unsure about anything, ask. Don't invent.

## 8. Getting help

- **Confused about a decision?** Check the journal in `PROJECT_STATE_FOR_AI.md` — search for the relevant cluster ID
- **Lint failing on a rule that seems wrong?** See `LINT_RULES.md` for justification; propose update via PR
- **Don't know which cluster to work on?** Pick the lowest-numbered open P0 in `UX_AUDIT_2026-05-25.md` §5
- **Stuck on env setup?** Re-read §3 above; if still stuck, the issue is likely outside Fynlo (Android SDK install, Java version)

---

## 9. After your first month

If you've been working on Fynlo for a month, you should:
- Have shipped at least one PR
- Know all 22 cluster IDs and what they're about (rough understanding)
- Recognize all 5 archetypes from `DESIGN_SYSTEM.md` §1
- Be able to spot anti-patterns from `DESIGN_SYSTEM.md` §16 in code

If not, the documentation is failing you — propose updates.

---

## 10. The one rule

**Read the docs. They exist because mistakes here cost real money. The 60 minutes you spend reading saves the 60 hours fixing a regression you'd otherwise have introduced.**
