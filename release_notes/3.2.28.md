## 1. What changed under the hood — Stage 3 only

Stages 1 and 2 already documented in `release_notes/3.2.25.md` and `release_notes/3.2.27.md`. This file covers Stage 3 — the final stage closing the cluster.

### Audit fixes landed

- **§C12 #5 — "Standardize Lent vs Owed rows to identical visual structure."** `LendingCard` and `DebtCard` now have the same shape: 40-dp circular avatar (Person icon for Lent, CreditCard for Owed) → name + optional OVERDUE chip → due-date or lent-date sub-line → outstanding amount + "Outstanding" label → chevron. Same paddings, same typography, same colours (Lent = Emerald, Owed = SemanticRed).

- **§C12 #6 — "Borrower row: icon + name + amount + chevron. One row tap → detail screen. NO action icons in row."** Both rows are now click-only — entire row is `Modifier.clickable { onClick() }` that calls `onNavigateToDetail(id)`. Every per-row action callback removed:
  - `LendingCard` lost: `onDelete`, `onEdit`, `onCollect`, `onDefault`, `onWriteOff`. Previously had 6 callbacks; now 1 (`onClick`).
  - `DebtCard` lost: `onEdit`, `onDelete`, `onPay`. Previously had 3 callbacks; now 1 (`onClick`).
  - The 70-line in-row WhatsApp message builder, the SMS builder, the "Share loan summary" button, the "Add Phone" hint, the MoreVert dropdown menu, the inline "Collect Payment" button, the inline "Mark as Defaulted / NPA" toggle, the inline status Badge, the inline interest accrual readout, the "Days Elapsed / Per Day Interest / Paid So Far" strip, the "Total Outstanding" duplicate inside each card, and the SI+CI portion split — **all of it gone from the list rows.** The row's job is to be scannable; the detail screen's job is to host actions.

- **§C12 #7 — "Detail screen has all actions (Pay / Collect / WhatsApp / SMS / Edit / Delete) as proper buttons."**
  - `CustomerDetailScreen` (Lent detail) already had Edit / Delete / Export-PDF in the TopBar and a full-width Collect Payment button. Added: Send Reminder (OutlinedButton, primary tier), Mark NPA / Restore Active (toggles per status), Write Off (only when outstanding > 0). The Mark NPA + Write Off pair sits in a Row at the bottom of the action stack as secondary actions.
  - **New `DebtDetailScreen`** (Owed detail) — mirrors `CustomerDetailScreen` structure. TopBar has Edit + Delete; body has hero outstanding number + Make Payment primary button + Payment History list + Notes card. Wired into the nav graph at route `debt/{debtId}`; the new `onNavigateToDebtDetail` prop on `LoansHubScreen` flows from `Navigation.kt` → `LoansHubScreen` → `DebtScreen.onNavigateToDetail`.

- **§C12 #8 — "Consolidate WhatsApp + SMS into single 'Send reminder' action with picker."** The Send Reminder button on `CustomerDetailScreen` opens an `AlertDialog` channel-picker with two TextButton actions: **WhatsApp** (Call icon) and **SMS** (Sms icon), plus Close. The smart-message builder is lifted into the dialog scope so the dispatched intent carries the same overdue-aware text that the old per-row icons did:
  - Overdue path: "your loan repayment is *N days overdue*", lists principal + accrued interest + paid + outstanding, ends with "Interest is adding ~₹X/day."
  - Friendly path: "friendly reminder about your outstanding loan (due in N days)", same loan summary, ends "Kindly arrange repayment at your convenience."
  - SMS variant is the trimmed-essentials version of the same message.
  - If the borrower has no phone number on file, the dialog shows a hint ("Tap Edit to add one before sending reminders") and hides both channel buttons.

### Deferred from §C12 — staying open as a Task

- **§C12 #4** — "column-header sort affordance instead." Stage 2 dropped the user-toggleable Name / Date sort modes in favour of a fixed default sort (overdue-first, then amount-desc). The audit's preferred fix is a column-header tap, but in the current list-row layout there are no column headers to tap. This is deferred — if user demand surfaces for explicit sort choices, can revisit as a follow-up.

## 2. Migration / data-shape disclosure

**None.** Pure UI restructure. No DB schema change, no JSON backup change, no DataStore key added. Behavioural compat:

- Old per-row "Collect" / "Mark NPA" / "Edit" / "Delete" tap targets are gone from the list. Users who used to tap straight into these from the list now tap the row → detail screen → tap the equivalent labelled button. One extra tap, but every action discoverable in one place with clear labels.
- WhatsApp icon and SMS icon are gone from rows. Send Reminder lives behind a single button in detail that opens a picker. Same dispatching code paths underneath (WhatsApp URL or `ACTION_SENDTO` for SMS).
- Add Phone shortcut from no-phone borrowers is gone from rows. Same affordance lives in the detail screen via the Edit action in TopBar.
- DebtScreen now has a per-row tap target; previously rows had per-card action buttons and no tap-to-detail. New `debt/{debtId}` route in nav graph.

## 4. What 3.2.28 does NOT include

For visibility — explicitly **not** part of this milestone:

- **Column-header sort affordance** (audit §C12 #4 deferred from Stage 2). The list rows don't have column headers in the current layout; a deliberate column-header design would be its own work. Tracking only if user demand surfaces.
- **Pay-debt-from-list shortcut.** Old DebtCard had a "Pay Instalment" button inline; that's gone with the row simplification. Users now tap row → detail → Make Payment. One extra tap; matches the Lent flow.
- **DebtDetailScreen feature parity with Lent.** Lent detail has Send Reminder + NPA + Write Off + PDF export; Debt detail doesn't, because (a) you don't message yourself to remind to pay debts, (b) NPA/Write-Off are receivable-side concepts, (c) PDF statement for debts is a deferred item. Asymmetric on purpose.
- **Tests for the new UI.** No Compose UI tests added; blocked on INF05 (instrumented tests in CI). Data layer for borrowers/debts is fully covered by existing data-integrity gate.
- **C13–C28 remaining clusters.** Per release-cadence ADR they all ship together at C22 closure.
